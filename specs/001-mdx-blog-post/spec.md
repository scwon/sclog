# Feature Specification: MDX 기반 블로그 포스트 시스템

**Feature Branch**: `001-mdx-blog-post`
**Created**: 2025-12-06
**Status**: Draft
**Input**: User description: "MDX 기반의 블로그 포스트 시스템 개발"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 블로그 글 작성 및 발행 (Priority: P1)

블로그 작성자는 MDX 형식으로 글을 작성하고, 빌드 시 자동으로 블로그 페이지가 생성되어 독자들이 볼 수 있다.

**Why this priority**: 블로그의 핵심 기능. 글이 없으면 블로그가 존재할 수 없다.

**Independent Test**: MDX 파일을 작성하고 빌드하면 해당 글이 웹페이지로 접근 가능한지 확인

**Acceptance Scenarios**:

1. **Given** 작성자가 MDX 파일을 콘텐츠 디렉토리에 생성했을 때, **When** 사이트를 빌드하면, **Then** 해당 글이 고유 URL로 접근 가능한 페이지가 된다
2. **Given** MDX 파일에 제목, 날짜, 태그 등의 메타데이터가 있을 때, **When** 페이지가 렌더링되면, **Then** 메타데이터가 적절한 위치에 표시된다
3. **Given** MDX 파일에 코드 블록이 있을 때, **When** 페이지가 렌더링되면, **Then** 코드가 구문 강조(syntax highlighting)와 함께 표시된다

---

### User Story 2 - 블로그 목록 페이지 (Priority: P2)

방문자는 블로그 메인 페이지에서 모든 글의 목록을 볼 수 있고, 최신 글이 먼저 표시된다.

**Why this priority**: 독자가 글을 발견하고 탐색할 수 있는 진입점

**Independent Test**: 블로그 메인 페이지 접속 시 발행된 모든 글이 날짜순으로 정렬되어 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 여러 개의 블로그 글이 존재할 때, **When** 블로그 목록 페이지에 접속하면, **Then** 모든 글이 최신순으로 정렬되어 표시된다
2. **Given** 각 글에 제목, 발행일, 요약이 있을 때, **When** 목록을 보면, **Then** 각 글의 제목, 발행일, 요약(또는 발췌)이 표시된다
3. **Given** 글 목록에서 특정 글을 클릭할 때, **When** 클릭하면, **Then** 해당 글의 상세 페이지로 이동한다

---

### User Story 3 - 태그 기반 분류 (Priority: P3)

방문자는 태그를 통해 관심 있는 주제의 글만 필터링하여 볼 수 있다.

**Why this priority**: 글이 많아질수록 분류와 탐색이 중요해짐

**Independent Test**: 특정 태그 클릭 시 해당 태그가 있는 글만 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 글에 태그가 지정되어 있을 때, **When** 글 상세 페이지를 보면, **Then** 해당 글의 태그가 표시되고 클릭 가능하다
2. **Given** 특정 태그를 클릭할 때, **When** 태그 페이지로 이동하면, **Then** 해당 태그가 있는 글만 목록으로 표시된다
3. **Given** 태그 목록 페이지가 있을 때, **When** 접속하면, **Then** 사용된 모든 태그와 각 태그의 글 개수가 표시된다

---

### User Story 4 - SEO 최적화 (Priority: P2)

검색 엔진이 블로그 글을 잘 인덱싱할 수 있도록 적절한 메타 태그와 구조화된 데이터가 제공된다.

**Why this priority**: 블로그의 발견 가능성과 트래픽에 직접적 영향

**Independent Test**: 각 페이지의 HTML 소스에서 meta 태그와 Open Graph 태그가 올바르게 생성되는지 확인

**Acceptance Scenarios**:

1. **Given** 블로그 글이 렌더링될 때, **When** HTML을 검사하면, **Then** title, description, og:title, og:description, og:image 태그가 존재한다
2. **Given** 블로그 글의 메타데이터에 설명이 있을 때, **When** 검색 결과에 표시되면, **Then** 해당 설명이 스니펫으로 표시된다
3. **Given** 사이트맵이 생성될 때, **When** 검색 엔진이 크롤링하면, **Then** 모든 블로그 글의 URL이 포함되어 있다

---

### Edge Cases

- 메타데이터가 없거나 불완전한 MDX 파일이 있을 때 → 기본값으로 대체하거나 빌드 경고 표시
- 동일한 슬러그(URL)를 가진 글이 여러 개 있을 때 → 빌드 에러로 알림
- 태그가 없는 글이 있을 때 → "미분류" 또는 태그 없이 정상 표시
- 매우 긴 글(10,000자 이상)이 있을 때 → 정상 렌더링 및 읽기 시간 표시
- 미래 날짜로 설정된 글이 있을 때 → 해당 날짜까지 목록에서 숨김 (예약 발행)

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 MDX 형식의 파일을 블로그 글로 인식하고 렌더링해야 한다
- **FR-002**: 각 블로그 글은 고유한 URL(슬러그)을 가져야 한다
- **FR-003**: 블로그 글에는 제목, 발행일, 요약, 태그 등의 메타데이터를 포함할 수 있어야 한다
- **FR-004**: 시스템은 코드 블록에 구문 강조(syntax highlighting)를 적용해야 한다
- **FR-005**: 시스템은 모든 블로그 글의 목록을 날짜순으로 정렬하여 표시해야 한다
- **FR-006**: 시스템은 태그별로 글을 필터링할 수 있는 기능을 제공해야 한다
- **FR-007**: 각 페이지는 SEO에 필요한 메타 태그(title, description, og:*)를 포함해야 한다
- **FR-008**: 시스템은 사이트맵을 자동 생성해야 한다
- **FR-009**: 글 본문에서 이미지, 링크, 테이블 등 마크다운 요소가 정상 렌더링되어야 한다
- **FR-010**: 미래 날짜의 글은 해당 날짜까지 목록에서 숨겨져야 한다 (예약 발행)

### Key Entities

- **BlogPost**: 개별 블로그 글을 나타냄. 제목, 슬러그, 발행일, 요약, 본문 내용, 태그 목록, 대표 이미지(선택) 포함
- **Tag**: 글의 분류를 위한 키워드. 이름과 해당 태그를 가진 글 목록 연결
- **PostMetadata**: 글의 frontmatter 정보. 제목, 날짜, 태그, 요약, 대표 이미지 URL 등

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작성자가 MDX 파일을 추가하면 5분 이내에 빌드 및 배포가 완료되어 글이 공개된다
- **SC-002**: 모든 블로그 페이지의 Lighthouse SEO 점수가 90점 이상이다
- **SC-003**: 블로그 목록 페이지가 1초 이내에 로드된다 (LCP 기준)
- **SC-004**: 100개 이상의 글이 있어도 목록 페이지 로드 시간이 2초를 초과하지 않는다
- **SC-005**: 방문자가 원하는 글을 3번의 클릭 이내에 찾을 수 있다 (목록 → 태그/검색 → 글)
- **SC-006**: 모든 글이 검색 엔진에 1주일 이내에 인덱싱된다 (사이트맵 제출 후)
