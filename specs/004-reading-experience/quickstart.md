# Quickstart: 블로그 읽기 경험 개선

**Date**: 2025-12-07
**Feature**: 004-reading-experience

## 개발 환경 설정

```bash
# 저장소 클론 (이미 완료된 경우 생략)
git clone <repository-url>
cd sclog

# 브랜치 체크아웃
git checkout 004-reading-experience

# 의존성 설치
pnpm install

# 개발 서버 실행
pnpm dev
```

---

## 기능 확인 방법

### 1. 읽기 시간 (이미 구현됨)

1. http://localhost:4321/blog 접속
2. 아무 블로그 글 클릭
3. 제목 아래 "N분 읽기" 표시 확인

### 2. 목차 (TOC)

1. h2, h3 헤딩이 3개 이상 있는 글 열기
2. 데스크톱(768px 이상): 우측에 목차 표시 확인
3. 목차 항목 클릭 → 해당 섹션으로 스크롤
4. 스크롤 시 현재 섹션 하이라이트 변경 확인
5. 모바일(768px 미만): 목차 숨김 확인

### 3. 코드 복사 버튼

1. 코드 블록이 있는 글 열기
2. 코드 블록 우상단 언어 라벨 확인 (예: "typescript")
3. 복사 버튼 클릭
4. "복사됨" 피드백 확인 (2초 후 원복)
5. 다른 곳에 붙여넣기하여 복사 확인

### 4. 스크롤 진행률 바

1. 긴 글 열기
2. 페이지 상단에 진행률 바 확인 (0% 상태)
3. 스크롤하면서 바가 채워지는지 확인
4. 끝까지 스크롤 → 100% 확인

### 5. 맨 위로 버튼

1. 페이지 상단에서 버튼 숨김 상태 확인
2. 아래로 스크롤 (300px 이상)
3. 우하단에 버튼 나타남 확인
4. 버튼 클릭 → 부드럽게 상단으로 이동
5. 상단 도착 후 버튼 숨김 확인

## 문제 해결

### TOC가 표시되지 않음
- 헤딩 개수 확인 (3개 이상 필요)
- 헤딩 레벨 확인 (h2, h3만 포함)

### 코드 복사가 안 됨
- HTTPS 환경인지 확인 (Clipboard API 요구사항)
- 브라우저 콘솔에서 에러 확인

### 진행률 바가 보이지 않음
- 페이지 높이가 뷰포트보다 큰지 확인
- CSS `position: fixed` 스타일 확인
