# Feature Specification: 디자인 시스템 적용

**Feature Branch**: `002-design-system`
**Created**: 2025-12-06
**Status**: Draft
**Input**: User description: "디자인 시스템 적용"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 라이트 모드 기본 테마 적용 (Priority: P1)

사용자가 블로그에 처음 접속하면 밝고 깔끔한 라이트 모드 디자인으로 콘텐츠를 편안하게 읽을 수 있다. 브랜드 컬러(코랄/피치 톤)가 강조 요소에 적용되어 SCLOG만의 정체성을 느낄 수 있다.

**Why this priority**: 디자인 시스템의 기반이 되는 핵심 요소로, 이것이 없으면 다른 기능 구현이 불가능하다. 첫인상을 결정하는 가장 중요한 요소.

**Independent Test**: 블로그 메인 페이지와 글 상세 페이지에서 라이트 테마가 올바르게 적용되었는지 시각적으로 확인

**Acceptance Scenarios**:

1. **Given** 사용자가 처음 블로그에 접속, **When** 페이지가 로드되면, **Then** 흰색 배경에 어두운 텍스트의 라이트 모드로 표시된다
2. **Given** 라이트 모드 상태, **When** 링크나 강조 요소를 확인하면, **Then** 브랜드 컬러 rgb(233, 172, 159)가 적용되어 있다
3. **Given** 라이트 모드 상태, **When** "SCLOG" 로고를 확인하면, **Then** 대문자로 표기되고 브랜드 컬러가 적용되어 있다
4. **Given** 라이트 모드 상태, **When** 폰트를 확인하면, **Then** Roboto(영문)와 Noto Sans KR(한글)이 적용되어 있다

---

### User Story 2 - 다크 모드 스위칭 (Priority: P1)

사용자가 테마 토글 버튼을 클릭하면 다크 모드로 전환되어 어두운 환경에서도 편안하게 콘텐츠를 읽을 수 있다. 테마 선택은 저장되어 다음 방문 시에도 유지된다.

**Why this priority**: 라이트 모드와 함께 필수적인 사용자 경험 요소. 많은 사용자가 다크 모드를 선호하며, 접근성 측면에서도 중요.

**Independent Test**: 테마 토글 버튼 클릭 시 다크 모드로 전환되고, 새로고침 후에도 선택이 유지되는지 확인

**Acceptance Scenarios**:

1. **Given** 라이트 모드 상태, **When** 테마 토글 버튼을 클릭하면, **Then** 다크 모드로 전환된다 (어두운 배경, 밝은 텍스트)
2. **Given** 다크 모드 상태, **When** 테마 토글 버튼을 클릭하면, **Then** 라이트 모드로 전환된다
3. **Given** 다크 모드를 선택한 상태, **When** 페이지를 새로고침하면, **Then** 다크 모드가 유지된다
4. **Given** 다크 모드 상태, **When** 브랜드 컬러 요소를 확인하면, **Then** 브랜드 컬러가 다크 모드에서도 동일하게 적용된다

---

### User Story 3 - 시스템 테마 자동 감지 (Priority: P2)

사용자의 운영체제 테마 설정을 자동으로 감지하여 첫 방문 시 적절한 테마로 표시한다. 사용자가 수동으로 테마를 변경하면 해당 선택이 우선된다.

**Why this priority**: 사용자 편의성을 높이는 부가 기능. 핵심 테마 기능이 완성된 후 추가하면 된다.

**Independent Test**: 시스템 다크 모드 설정 상태에서 첫 방문 시 다크 모드로 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 시스템이 다크 모드로 설정된 상태에서 첫 방문, **When** 페이지가 로드되면, **Then** 자동으로 다크 모드로 표시된다
2. **Given** 시스템이 라이트 모드로 설정된 상태에서 첫 방문, **When** 페이지가 로드되면, **Then** 라이트 모드로 표시된다
3. **Given** 사용자가 수동으로 테마를 변경한 상태, **When** 시스템 테마가 변경되어도, **Then** 사용자의 수동 선택이 유지된다

---

### User Story 4 - 일관된 컴포넌트 스타일 (Priority: P2)

모든 UI 컴포넌트(버튼, 카드, 태그, 입력 필드 등)가 디자인 시스템에 정의된 스타일을 일관되게 적용하여 통일된 시각적 경험을 제공한다.

**Why this priority**: 전체적인 디자인 완성도를 높이는 요소. 기본 테마 적용 후 컴포넌트별 세부 스타일을 적용한다.

**Independent Test**: 블로그 전체를 탐색하며 모든 컴포넌트가 일관된 스타일을 가지는지 확인

**Acceptance Scenarios**:

1. **Given** 블로그 카드 컴포넌트, **When** 화면에 표시되면, **Then** 8px 라운딩, 정의된 테두리 색상이 적용된다
2. **Given** 태그 컴포넌트, **When** 화면에 표시되면, **Then** pill 형태(9999px 라운딩), 브랜드 컬러 배경이 적용된다
3. **Given** 라이트/다크 모드 전환 시, **When** 모든 컴포넌트를 확인하면, **Then** 각 모드에 맞는 색상이 일관되게 적용된다

---

### Edge Cases

- 사용자의 브라우저가 localStorage를 지원하지 않는 경우: 시스템 테마를 따르고, 세션 동안만 수동 선택 유지
- 테마 전환 중 콘텐츠가 깜빡이는 현상(FOUC): 페이지 로드 전 테마를 먼저 적용하여 방지
- 구형 브라우저에서 CSS 변수를 지원하지 않는 경우: 라이트 모드 폴백 적용
- 사용자가 시스템 테마를 "자동(시간대별)"으로 설정한 경우: prefers-color-scheme 미디어 쿼리로 실시간 대응

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 라이트 모드를 기본 테마로 표시해야 한다
- **FR-002**: 시스템은 다크 모드와 라이트 모드 간 전환을 지원해야 한다
- **FR-003**: 사용자는 테마 토글 버튼을 통해 테마를 변경할 수 있어야 한다
- **FR-004**: 시스템은 사용자의 테마 선택을 저장하고 다음 방문 시 복원해야 한다
- **FR-005**: 시스템은 사용자의 시스템 테마 설정(prefers-color-scheme)을 감지해야 한다
- **FR-006**: "SCLOG" 브랜드명은 모든 위치에서 대문자로 표기되어야 한다
- **FR-007**: 브랜드 컬러 rgb(233, 172, 159)가 강조 요소에 적용되어야 한다
- **FR-008**: Roboto 폰트(영문)와 Noto Sans KR 폰트(한글)가 적용되어야 한다
- **FR-009**: 테마 전환 시 깜빡임(FOUC) 없이 부드럽게 전환되어야 한다
- **FR-010**: 모든 UI 컴포넌트는 정의된 디자인 토큰을 사용해야 한다

### Key Entities

- **Theme**: 테마 상태 (light, dark, system), 사용자 선택 여부
- **Design Token**: 색상 변수, 타이포그래피 변수, 레이아웃 변수를 포함하는 스타일 시스템

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 테마를 1초 이내에 전환할 수 있다
- **SC-002**: 페이지 새로고침 후 테마 선택이 100% 유지된다
- **SC-003**: 테마 전환 시 시각적 깜빡임이 발생하지 않는다
- **SC-004**: 모든 페이지에서 브랜드 컬러와 폰트가 일관되게 적용된다
- **SC-005**: 라이트/다크 모드 모두에서 텍스트 가독성이 WCAG AA 기준을 충족한다
- **SC-006**: 첫 방문 사용자의 시스템 테마가 90% 이상 정확히 감지된다

## Assumptions

- 브라우저 localStorage 지원을 가정 (미지원 시 세션 스토리지 또는 쿠키 폴백)
- Google Fonts CDN을 통한 폰트 로드
- CSS Custom Properties(변수) 지원 브라우저 대상 (IE11 미지원)
- 테마 전환 애니메이션은 150ms 이내로 제한하여 성능 영향 최소화
