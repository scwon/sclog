# Feature Specification: 포트폴리오 홈페이지

**Feature Branch**: `003-portfolio-home`
**Created**: 2025-12-07
**Status**: Draft
**Input**: 첫 페이지 개발 - 포트폴리오 페이지로 자기소개, 소셜 링크(GitHub, LinkedIn, Instagram), 이메일, 이력, 최근 글 포함

## User Scenarios & Testing _(mandatory)_

### User S/ptory 1 - 첫인상 확인 (Priority: P1)

방문자가 SCLOG 홈페이지에 처음 접속하여 블로그 주인이 누구인지 빠르게 파악한다.

**Why this priority**: 첫인상이 가장 중요하며, 방문자가 3초 내에 "이 사람이 누구인지"를 이해해야 이탈하지 않음

**Independent Test**: 홈페이지에 접속하여 자기소개 섹션만 확인해도 블로그 주인의 정체성을 파악할 수 있음

**Acceptance Scenarios**:

1. **Given** 방문자가 SCLOG 홈페이지에 접속했을 때, **When** 페이지가 로드되면, **Then** 프로필 사진/아바타, 이름, 한 줄 소개가 화면 상단에 표시된다
2. **Given** 방문자가 홈페이지를 보고 있을 때, **When** 자기소개 섹션을 확인하면, **Then** 간단한 자기소개 문구(2-3문장)를 읽을 수 있다

---

### User Story 2 - 연락처 및 소셜 링크 확인 (Priority: P2)

방문자가 블로그 주인에게 연락하거나 다른 플랫폼에서 팔로우하고 싶을 때 쉽게 연결할 수 있다.

**Why this priority**: 네트워킹과 기회 연결을 위해 연락 수단이 명확해야 함

**Independent Test**: 소셜 링크 아이콘을 클릭하여 각 플랫폼으로 이동할 수 있음

**Acceptance Scenarios**:

1. **Given** 방문자가 홈페이지에 있을 때, **When** 소셜 링크 영역을 확인하면, **Then** GitHub, LinkedIn, Instagram 아이콘이 표시된다
2. **Given** 방문자가 소셜 아이콘을 클릭했을 때, **When** 링크가 열리면, **Then** 새 탭에서 해당 플랫폼의 프로필 페이지가 열린다
3. **Given** 방문자가 이메일로 연락하고 싶을 때, **When** 이메일 링크/아이콘을 클릭하면, **Then** 이메일 클라이언트가 열리거나 이메일 주소가 복사된다

---

### User Story 3 - 경력/이력 확인 (Priority: P3)

채용 담당자나 협업 희망자가 블로그 주인의 경력과 기술 역량을 파악한다.

**Why this priority**: 포트폴리오의 핵심 목적이지만, 먼저 관심을 가져야 이력을 확인함

**Independent Test**: 이력 섹션을 스크롤하여 경력 정보를 확인할 수 있음

**Acceptance Scenarios**:

1. **Given** 방문자가 홈페이지를 스크롤할 때, **When** 이력 섹션에 도달하면, **Then** 주요 경력 사항이 시간순으로 표시된다
2. **Given** 방문자가 이력을 확인할 때, **When** 각 경력 항목을 보면, **Then** 회사/프로젝트명, 역할, 기간이 표시된다

---

### User Story 4 - 최근 글 탐색 (Priority: P4)

방문자가 블로그 주인의 관심사와 전문성을 글을 통해 파악하고, 관심 있는 글로 이동한다.

**Why this priority**: 블로그 콘텐츠로의 자연스러운 유입 경로 제공

**Independent Test**: 최근 글 섹션에서 글 제목을 클릭하여 해당 글로 이동할 수 있음

**Acceptance Scenarios**:

1. **Given** 방문자가 홈페이지를 스크롤할 때, **When** 최근 글 섹션에 도달하면, **Then** 최근 작성된 블로그 글 목록(3-5개)이 표시된다
2. **Given** 방문자가 최근 글 목록을 보고 있을 때, **When** 글 제목을 클릭하면, **Then** 해당 블로그 글 페이지로 이동한다
3. **Given** 방문자가 더 많은 글을 보고 싶을 때, **When** "모든 글 보기" 링크를 클릭하면, **Then** 블로그 글 목록 페이지로 이동한다

---

### Edge Cases

- 소셜 링크 중 일부가 없는 경우: 해당 아이콘은 표시하지 않음
- 블로그 글이 하나도 없는 경우: "아직 작성된 글이 없습니다" 메시지 표시
- 이력 정보가 없는 경우: 이력 섹션 전체를 숨김
- 이미지 로딩 실패 시: 대체 아바타(이니셜 또는 기본 아이콘) 표시

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템 MUST 프로필 섹션에 사진/아바타, 이름, 한 줄 소개를 표시한다
- **FR-002**: 시스템 MUST 자기소개 문구(2-3문장)를 표시한다
- **FR-003**: 시스템 MUST 소셜 링크(GitHub, LinkedIn, Instagram)를 아이콘 형태로 표시한다
- **FR-004**: 시스템 MUST 이메일 연락 방법을 제공한다 (mailto 링크 또는 복사 기능)
- **FR-005**: 시스템 MUST 모든 외부 링크를 새 탭에서 열어야 한다
- **FR-006**: 시스템 MUST 이력/경력 정보를 시간순으로 표시한다
- **FR-007**: 시스템 MUST 최근 블로그 글 3-5개를 제목, 날짜와 함께 표시한다
- **FR-008**: 시스템 MUST 각 블로그 글 항목 클릭 시 해당 글 페이지로 이동한다
- **FR-009**: 시스템 MUST "모든 글 보기" 링크를 제공하여 블로그 목록 페이지로 이동할 수 있게 한다
- **FR-010**: 시스템 MUST 헌법의 Design System 원칙(브랜드 컬러, 타이포그래피, 레이아웃)을 준수한다
- **FR-011**: 시스템 MUST 라이트/다크 테마 모두에서 올바르게 표시된다
- **FR-012**: 시스템 MUST 모바일과 데스크톱에서 반응형으로 동작한다

### Key Entities

- **Profile**: 블로그 주인의 개인 정보 (이름, 사진, 소개, 이메일)
- **SocialLink**: 외부 플랫폼 연결 정보 (플랫폼 종류, URL, 아이콘)
- **Career**: 경력 정보 (회사/프로젝트명, 역할, 기간, 설명)
- **BlogPost**: 블로그 글 (기존 MDX 콘텐츠 시스템 활용)

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 방문자가 페이지 로드 후 3초 내에 블로그 주인의 정체성(이름, 역할)을 파악할 수 있다
- **SC-002**: 모든 소셜 링크가 클릭 시 1초 내에 새 탭에서 올바른 페이지로 이동한다
- **SC-003**: 최근 글 섹션에서 글 클릭 시 해당 블로그 글 페이지로 올바르게 이동한다
- **SC-004**: 페이지가 모바일(320px)부터 데스크톱(1920px)까지 모든 화면 크기에서 깨지지 않고 표시된다
- **SC-005**: Core Web Vitals 기준 충족 (LCP < 2.5s, CLS < 0.1)

## Assumptions

- 프로필 정보(이름, 소개, 이메일, 소셜 링크)는 하드코딩 또는 설정 파일로 관리한다
- 이력 정보는 정적 데이터로 관리한다 (별도 CMS 연동 없음)
- 최근 글은 기존 MDX 기반 블로그 시스템에서 가져온다
- 소셜 링크는 GitHub, LinkedIn, Instagram 3개로 시작하며, 추후 확장 가능하다
